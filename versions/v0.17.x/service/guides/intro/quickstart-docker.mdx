**CyborgDB** can be deployed as a standalone microservice using Docker. This allows you to run a fully self-contained encrypted vector search service on your own infrastructure with minimal setup. The service exposes a REST API for integration with any stack.

<Tip>Looking to use the Python service? Check out our [Python Quickstart Guide](./quickstart-python).</Tip>

## Overview
The Docker service is ideal for teams looking to self-host CyborgDB in cloud, on-prem, or containerized environments.

This guide takes you from a clean machine to your first encrypted query in one sitting — no API key to obtain, and no database to configure.

<Steps>
    <Step title="Pull the Docker Image">
        The CyborgDB service is available as a Docker image. Pull it from Docker Hub:

        ```bash
        docker pull cyborginc/cyborgdb-service:latest
        ```

        This image contains everything you need to run the CyborgDB service, including all dependencies and configurations.
    </Step>

    <Step title="Run the Service">
        Start the service with its built-in **standalone** store — no API key and no database configuration required:

        ```bash
        docker run -it --rm -p 127.0.0.1:8000:8000 \
          -v cyborgdb_data:/app/cyborgdb_data \
          cyborginc/cyborgdb-service:latest
        ```

        <Warning>The example binds to `127.0.0.1` so the service is reachable only from your machine. The service has no authentication layer — do not publish the port on `0.0.0.0` or expose it to the public internet.</Warning>

        <Note>The standalone store is for evaluation and local development. For production backends (PostgreSQL, Redis, S3), HTTPS, and Docker Compose setups, see [Environment Variables](../advanced/env-vars) and [Backing Stores](../../../intro/backing-stores).</Note>
    </Step>

    <Step title="Verify the Service">
        Confirm the service is healthy:

        ```bash
        curl http://localhost:8000/v1/health
        ```

        You should see a response indicating the service is healthy and ready to accept requests. You can also explore the interactive API documentation at [http://localhost:8000/v1/docs](http://localhost:8000/v1/docs).
    </Step>

    <Step title="Install the Client SDK">
        With the service running, install the Python client SDK in your application environment:

        ```bash
        pip install cyborgdb
        ```
    </Step>

    <Step title="Connect to the Service">
        Create a client pointing at your running service:

        ```python
        from cyborgdb import Client

        # Connect to the local service — no API key required
        client = Client(base_url='http://localhost:8000')
        ```
    </Step>

    <Step title="Create an Encrypted Index">
        Generate an encryption key and create an index:

        ```python
        # ... Continuing from the previous step

        # Generate a 32-byte encryption key for the index
        index_key = client.generate_key()

        # Create an encrypted index
        index = client.create_index(
            index_name="my_index",
            index_key=index_key
        )
        ```

        <Note>
        The **encryption key** protects your index contents and is separate from how you connect to the service. Store it securely, outside your source code — if you lose it, the index cannot be decrypted, and if you commit it to version control, you expose your data. This is not an account or API key; the service itself requires no key to run.
        </Note>
    </Step>

    <Step title="Add Items">
        Add items to the encrypted index with `upsert`:

        ```python
        # ... Continuing from the previous step

        items = [
            {"id": "item_1", "vector": [0.1, 0.2, 0.3, 0.4], "contents": "Hello!"},
            {"id": "item_2", "vector": [0.5, 0.6, 0.7, 0.8], "contents": "Bonjour!"},
            {"id": "item_3", "vector": [0.9, 0.10, 0.11, 0.12], "contents": "Hola!"}
        ]

        index.upsert(items)
        ```
    </Step>

    <Step title="Query the Index">
        Search the encrypted index for the nearest vectors:

        ```python
        # ... Continuing from the previous step

        query_vector = [0.1, 0.2, 0.3, 0.4]
        results = index.query(query_vectors=query_vector, top_k=3)

        # Results are a flat list of {"id", "distance"}, nearest first
        for result in results:
            print(f"ID: {result['id']}, Distance: {result['distance']}")
        ```

        Seeing IDs and distances printed means your service, index, and query are all working end-to-end.
    </Step>

    <Step title="Next Steps">
        You've gone from install to your first encrypted query. Explore further, or integrate from another language:

        <CodeGroup>
        ```bash JavaScript icon="js"
        npm install cyborgdb
        ```

        ```bash TypeScript icon="code"
        npm install cyborgdb
        ```

        ```bash Go icon="golang"
        go get github.com/cyborginc/cyborgdb-go
        ```
        </CodeGroup>

    </Step>
</Steps>

<Accordion title="Technical Specifications">

- **Base Image**: `python:3.11-slim`
- **Python Version**: 3.11
- **PyTorch**: CPU-optimized for maximum compatibility
- **Docker Image Size**: ~1.8GB
- **Platform**: linux/amd64, linux/arm64
- **Default Port**: 8000

</Accordion>

<Accordion title="Comparison with Python Service">

| Aspect | Python Service | Docker Service |
|--------|---------------|----------------|
| **Installation** | `pip install` | `docker run` |
| **Dependencies** | Managed by pip/conda | Bundled in container |
| **Resource Usage** | Lower overhead | Higher overhead |
| **Environment Isolation** | Python virtualenv | Container isolation |
| **Deployment Complexity** | Simple Python deployment | Container orchestration |
| **Configuration** | Environment variables/files | Environment variables |
| **Best For** | Development, Python-heavy workflows | Production, cloud deployment |

<Tip>Both approaches provide identical CyborgDB functionality. Choose based on your deployment preferences and infrastructure requirements.</Tip>

</Accordion>
