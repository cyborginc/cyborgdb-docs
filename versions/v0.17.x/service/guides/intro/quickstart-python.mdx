**CyborgDB** can be deployed as a Python service using pip installation. This approach provides a lightweight alternative to Docker, allowing you to run the CyborgDB service directly in your Python environment with full control over dependencies and configuration.

<Tip>Looking to use Docker? Check out our [Docker Quickstart Guide](./quickstart-docker).</Tip>

## Overview
The Python service is ideal for developers who prefer managing Python environments directly, need custom dependency management, or want to integrate CyborgDB service into existing Python applications. It provides the same REST API functionality as the Docker version.

This guide takes you from a clean machine to your first encrypted query in one sitting — no API key to obtain, and no database to configure.

<Steps>
    <Step title="Check Prerequisites">
        Ensure you have the required Python version and environment tools:

        - **Python**: 3.10, 3.11, 3.12, 3.13, or 3.14
        - **Package Manager**: conda (recommended) or pip with virtual environment

        <Note>While conda is mentioned in the build instructions, the wheel works with any Python environment manager including pip, pipenv, poetry, etc.</Note>
    </Step>

    <Step title="Install CyborgDB Service">
        Install the CyborgDB service package:

        <CodeGroup>
        ```bash Conda Environment (Recommended)
        # Create conda environment
        conda create -n cyborgdb-service python=3.12
        conda activate cyborgdb-service

        # Install the service
        pip install cyborgdb-service

        # If you want to use automatic embedding generation, install with:
        pip install cyborgdb-service[embeddings]
        ```

        ```bash Virtual Environment
        # Create a virtual environment
        python -m venv cyborgdb-env
        source cyborgdb-env/bin/activate

        # Install the service
        pip install cyborgdb-service
        ```

        ```bash System Python
        # Install directly (not recommended for production)
        pip install cyborgdb-service
        ```
        </CodeGroup>

        <Warning>Using conda or a virtual environment is strongly recommended to avoid dependency conflicts with other Python packages.</Warning>
    </Step>

    <Step title="Start the Service">
        Run the CyborgDB service using the installed command:

        ```bash
        cyborgdb-service
        ```

        No configuration is required to get started. The service launches with a built-in **standalone** store, so there is no database type or connection string to set. It displays startup information including the server URL (default: `http://localhost:8000`) and the available API endpoints.

        <Warning>The service listens on `localhost` by default and has no authentication layer. Keep it bound to localhost during development — do not expose the port to the public internet.</Warning>

        <Note>The standalone store is for evaluation and local development. For production backends (PostgreSQL, Redis, S3) and HTTPS, see [Environment Variables](../advanced/env-vars) and [Backing Stores](../../../intro/backing-stores).</Note>
    </Step>

    <Step title="Verify the Service">
        Once the service is running, confirm it's healthy:

        ```bash
        curl http://localhost:8000/v1/health
        ```

        You should see a response indicating the service is healthy and ready to accept requests. You can also explore the interactive API documentation at [http://localhost:8000/v1/docs](http://localhost:8000/v1/docs).
    </Step>

    <Step title="Install the Client SDK">
        With the service running, install the Python client SDK in your application environment:

        ```bash
        pip install cyborgdb
        ```
    </Step>

    <Step title="Connect to the Service">
        Create a client pointing at your running service:

        ```python
        from cyborgdb import Client

        # Connect to the local service — no API key required
        client = Client(base_url='http://localhost:8000')
        ```
    </Step>

    <Step title="Create an Encrypted Index">
        Generate an encryption key and create an index:

        ```python
        # ... Continuing from the previous step

        # Generate a 32-byte encryption key for the index
        index_key = client.generate_key()

        # Create an encrypted index
        index = client.create_index(
            index_name="my_index",
            index_key=index_key
        )
        ```

        <Note>
        The **encryption key** protects your index contents and is separate from how you connect to the service. Store it securely, outside your source code — if you lose it, the index cannot be decrypted, and if you commit it to version control, you expose your data. This is not an account or API key; the service itself requires no key to run.
        </Note>
    </Step>

    <Step title="Add Items">
        Add items to the encrypted index with `upsert`:

        ```python
        # ... Continuing from the previous step

        items = [
            {"id": "item_1", "vector": [0.1, 0.2, 0.3, 0.4], "contents": "Hello!"},
            {"id": "item_2", "vector": [0.5, 0.6, 0.7, 0.8], "contents": "Bonjour!"},
            {"id": "item_3", "vector": [0.9, 0.10, 0.11, 0.12], "contents": "Hola!"}
        ]

        index.upsert(items)
        ```
    </Step>

    <Step title="Query the Index">
        Search the encrypted index for the nearest vectors:

        ```python
        # ... Continuing from the previous step

        query_vector = [0.1, 0.2, 0.3, 0.4]
        results = index.query(query_vectors=query_vector, top_k=3)

        # Results are a flat list of {"id", "distance"}, nearest first
        for result in results:
            print(f"ID: {result['id']}, Distance: {result['distance']}")
        ```

        Seeing IDs and distances printed means your service, index, and query are all working end-to-end.
    </Step>

    <Step title="Next Steps">
        You've gone from install to your first encrypted query. Explore further, or integrate from another language:

        <CodeGroup>
        ```bash JavaScript icon="js"
        npm install cyborgdb
        ```

        ```bash TypeScript icon="code"
        npm install cyborgdb
        ```

        ```bash Go icon="golang"
        go get github.com/cyborginc/cyborgdb-go
        ```
        </CodeGroup>

    </Step>
</Steps>

<Accordion title="Comparison with Docker Service">

| Aspect | Python Service | Docker Service |
|--------|---------------|----------------|
| **Installation** | `pip install` | `docker run` |
| **Dependencies** | Managed by pip/conda | Bundled in container |
| **Resource Usage** | Lower overhead | Higher overhead |
| **Environment Isolation** | Python virtualenv | Container isolation |
| **Deployment Complexity** | Simple Python deployment | Container orchestration |
| **Configuration** | Environment variables/files | Environment variables |
| **Best For** | Development, Python-heavy workflows | Production, cloud deployment |

<Tip>Both approaches provide identical CyborgDB functionality. Choose based on your deployment preferences and infrastructure requirements.</Tip>

</Accordion>
